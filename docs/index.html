作品摘要說明

40% 完整性(教材內容與課程大綱內容結合情形)

本教材的主要是訓練學生「微處理機」及「微控制器」整合應用的技術，為「微控制器應用專題」及「介面技術」課程奠下基礎，其中最重要的包括1.微處理器I2C通訊介面、2. MQTT物聯網通訊協定應用、3. 微控制器 Arduino GPIO 、4. 期末微型專題 等，在課群內建立關聯。本教材混搭了「做中學」及「翻轉教室」的授課方式，希望學生能夠養成自學成長的學習態度，同時又能藉由同儕間程度相仿的語言來交流溝通授課內容，一來學生可以自我檢視並複習所學的專業知識，同時藉由共學成長來培養團隊合作精神，希望藉此正能量養成學生「如何學」的技能。

本教材包含：Arduino C/C++ 與 Python 基礎程式設計、GPIO與串列介面(UART 及 I2C)介紹、微控制器GPIO與串列介面程式設計、及微處理器串列介面程式設計。考量學生先備課程基礎及現有教學設備，我們選用目前廣泛用在教學場域的Arduino及樹梅派(Raspberry Pi)，搭配開源且合法免費使用的Arduino 整合開發環境，在微處理器端物聯網應用程式的開發，則使用資料科學及人工智慧常用的學習平台JupyterLab，方便將本課程所學的物聯網實作技術，銜接智慧物聯網的相關應用，本課程設定的專業訓練目標是具備整合應用實作能力的工程師，並非慣於使用Arduino及Raspberry Pi設計雛形的業餘自造者。

課程大綱：

本課程共24個單元，依單元主題分成：

  • 微處理器應用
    • Raspberry Pi開發環境簡介(單元 1~3)
    • Python 程式設計(單元 4~6、8、9、11~14)
  • 微控制器應用
    • Arduino UNO 程式基礎與GPIO應用 (單元 7、10)
    • IC 應用程式開發 (單元 17~20)
  • 微處理器/微控制器通訊介面
    • 通訊介面 I2C 簡介(單元 15、16)
    • 微控制器與微處理器間的資料傳輸 (單元 17~20)
  • 物聯網通訊協定
    • 物聯網通訊協定 MQTT 基礎(單元 21)
    • 物聯網(IoT)裝置實務設計(單元 22)
  • 期末微型專題
    • 物聯網(IoT)交通號誌控制系統 (單元 23、24)
      • Arduino UNO - 可程式化交通號誌系統
      • Raspberry Pi - 處理MQTT與IC的訊息傳遞
      • 物聯網控制介面 - 手機APP或Window應用程式

30% 創新性(教材原創性及貢獻度，說明該作品與類似作品之差異或優於類似教材部分)

本教材所有的範例程式及翻轉教室影音教材皆為本人『原創作品』，沒有使用任何書商提供的教材，因此也沒有指定教科書，所有教材內容都是本人搜尋國內外資料整理設計，引用到的外部參考資料也都有明確表列來源網址。教學方法與知識承載的媒體也做了大幅度的改變，全部使用E世代學生熟悉的 YouTube及雲端 Github 練習程式庫，有別於傳統的課堂授課方式，本課程融合了翻轉教室、做中學、及適性化教學等技巧，藉由漸進式調整的方式來因應近年來學生程度的轉變，不論是教學內容、授課方式、及教材呈現，都是針對南臺學生學習狀況客製化改良設計。

本教材適用於混搭「做中學」及「翻轉教室」的授課方式，鼓勵同儕間相互學習及團隊間的良性競合學習，激發學生的學習動機並有效提昇學習成效。課堂授課利用每週單元主題設計的課堂練習，引導學生經由「做中學」了解單元主題的專業知識，翻轉教室的部分就稍微複雜了點，我們會錄製翻轉教室影音教材，因考量學生大都沒有課前自主學習的習慣，課堂上進行翻轉活動前必須面授當週的單元主題，我們會適時的穿插「預覽」預習翻轉的影片，但不會在課堂上播放完整的影片，預覽的目的是讓學生知道目前所有的上課內容都有對應的影音教材。 課後本人會在印象還極為清晰的情況下，利用課堂上觀察到的上課狀況、學生提問、及助教回饋作為素材，設計「複習翻轉」影音教材腳本，這個課後錄製的影片目的在協助學生完成作業，複習翻轉的概念類似是自適應學習(Adaptive Learning)的輔助教學法，但因為時間較為緊迫無法針對個人做調整，只能夠以整個班級所觀察到的最大公約數，來設計錄製「複習翻轉」影音教材。 本教材之運用與「單晶片微處理機實務」課程授課方式說明如下: 錄製理論基礎及課堂實習翻轉教室影片，使學生能在進課堂前以最舒適的方式自主學習，藉此大幅降低課堂上因學習能力落差，所造成的低落學習意願及學習成效。本教材共24個單元(影片)約莫9個小時，依單元主題可細分成：Python 程式設計(單元 1~6、8、9、11~14)、微控制器 GPIO (單元 7、10)、微處理器通訊介面(單元 15~20)、物聯網通訊協定(單元 21、22)、期末上機考複習(單元 23)、期末微型專題(單元 24)等。詳細單元列表如下:

30% 學習成效(教材/教具輔助教學之運用情形，說明該作品對學生學習或教師教學之貢獻)

所有教材皆在實際授課過程中『動態調整』，依據課堂上觀察到的學生反應及作業繳交狀況，每週調整設計錄製影音輔助教學教材，所有課堂範例及練習，都是為學生客製化的全新設計，所以本教材內的所有教學影片及練習專案教具皆100%在課堂上使用，因為這是依每學期上課班級『客製化』規劃設計的教材，而這份翻轉教室影音教材也是本人今年榮獲『績優教學實踐計畫』的成果之一。

之前有上過這個專班的經驗，發覺這個班級的學習意願跟一般的班級差異不大，一開始直接使用去年為本課程設計的教材，幾次的作業後發覺學生無法跟上進度，主要是因為基礎程式設計能力的匱乏，雖然他們都有學過本課程所需要的兩種程式語言C和Python，但實際要他們用在本課程的作業上時，同學們的表現完全出乎我們的意料，能夠順利跟上進度的同學寥寥可數。因此我們把課程的內容做了大幅的調整，花了近乎一個月的時間幫同學們複習Python程式設計，也提供了翻轉教室的影音教材，協助同學提升程式設計能力到本課程的基本需求。 另一項教學策略上的調整是將微型專題單純化，同時提供較多的引導與協助，原本是希望同學能夠藉由微型專題製作過程，了解微處理器與微控制器之間的合作關係，但礙於大部分同學對微控制器Arduino UNO 設計極為陌生，因而把微控制器部分設計成固定的樣板提供學生使用，讓學生專注在微處理器Raspberry Pi上的應用程式設計，同時也提供現成的Android APP及Windows應用程式控制介面，協助輔導同學順利完成期末微型專題，專注於如何以I2C通訊介面與微控制器系統溝通，並運用物聯網通訊協定MQTT，將簡易交通號誌系統升級成物聯網(IoT) 雲端控制系統。 這個班級在最後一個月的表現，再次讓我和助教們跌破眼鏡，雖然我們做了很多的教學調整本就預期學生會有所反應，但我所觀察到的竟是幾乎全班同學都轉性了，似乎突然間覺得這一門課的內容變得有用有趣了，學期末三周學生們參與課堂練習的情況極為活絡，我都還得特別交代不要托住我的助教太久讓他們中午可以去吃飯，看來我們的學生是可以改變也會改變，但是身為老師的我們得不斷地去嘗試各式各類的教學法，平心而論「邊授課、邊觀察、邊調整」真的很累，但是看到學生們的正向回饋及改變，整個學期累積的疲憊似乎已散去不少。
